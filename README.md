Настройки и администрирование системы управления заказми для ресторанного бизнеса

Подразумевает под собой задачи такие как прием заказов, отправка заказа на кухню, мониторинг системы.

  Заказ – центральная часть, который имеет статусы такие как отправлен на кухню, в процессе, готов к выдаче
  Блюдо – располагается в меню. Имеет категорию (закуски, супы, основные и тд.), состав, себестоимость.
  Меню – создается несколько категорий для быстрого поиска блюда
  Для администратора создается мониторинг, где он проверяет работу серверов и добавлением новых функций.

Жизненный цикл заказа:
Официант принимает заказ, то есть добавляет в поле, где указан номер столика, а также общую стоимость заказа.
Потом происходит передача на кухню, где заказ принимает статус по готовке. Официант может отслеживать заказ по готовности и выручку заказов на сегодня.

Структура Серверной части:
    -	Модели данных
    -	Компоненты для взаимодействия с базой данных и redis 
    -	Схемы построения базы данных
    -	Маршруты для создания заказа 
    
Структура Клиентской части:
    -	поле меню, в котором есть все блюда сразу и по категориями для удобного выбора блюда
    -	поле с созданием нового заказа
    -	отправкой на кухню или же отчистить заказ
    -	поле с активностью заказами 
    -	статистикой (выручка, сколько заказов есть)
    -	поле с мониторингом для администратора
